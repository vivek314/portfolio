    **IMPORTANT: Use the Neon HTTP driver, NOT `pg` (node-postgres). TCP connections fail on Vercel serverless.**

    Create `src/db/index.ts` — the Drizzle client singleton:
    ```typescript
    // src/db/index.ts
    // Source pattern: https://neon.com/blog/nextjs-authentication-using-clerk-drizzle-orm-and-neon
    import { neon } from '@neondatabase/serverless'
    import { drizzle } from 'drizzle-orm/neon-http'
    import * as schema from './schema'

    if (!process.env.DATABASE_URL) {
      throw new Error('DATABASE_URL must be a Neon postgres connection string')
    }

    const sql = neon(process.env.DATABASE_URL)
    export const db = drizzle(sql, { schema })
    ```

    Create `src/db/schema.ts` — Phase 1 bootstrap schema with all tables needed through Phase 6.
    The schema must use Clerk user IDs as text foreign keys (NO local users table — Clerk owns user records):
    ```typescript
    // src/db/schema.ts
    import {
      pgTable,
      text,
      timestamp,
      uuid,
      jsonb,
      boolean,
    } from 'drizzle-orm/pg-core'

    // Submissions: user design code + scenario metadata
    // clerkUserId is the Clerk user.id string — used as FK across all tables
    // No DB-level FK constraint to a users table (Clerk manages users externally)
    export const submissions = pgTable('submissions', {
      id: uuid('id').defaultRandom().primaryKey(),
      clerkUserId: text('clerk_user_id').notNull(),
      scenarioId: text('scenario_id').notNull(),       // e.g. 'parking-lot'
      difficulty: text('difficulty').notNull(),         // 'easy' | 'medium' | 'hard'
      code: text('code').notNull(),
      createdAt: timestamp('created_at').defaultNow().notNull(),
      updatedAt: timestamp('updated_at').defaultNow().notNull(),
    })

    // EvaluationResults: LLM grading output per submission
    // criteriaResults is JSONB — stores CriterionResult[] array (see src/types/index.ts)
    export const evaluationResults = pgTable('evaluation_results', {
      id: uuid('id').defaultRandom().primaryKey(),
      submissionId: uuid('submission_id')
        .references(() => submissions.id, { onDelete: 'cascade' })
        .notNull(),
      criteriaResults: jsonb('criteria_results').notNull(), // CriterionResult[]
      overallPass: boolean('overall_pass').notNull(),
      llmModel: text('llm_model').notNull(),               // e.g. 'claude-3-5-sonnet-20241022'
      createdAt: timestamp('created_at').defaultNow().notNull(),
    })
    ```

    Create `drizzle.config.ts` at project root:
    ```typescript
    // drizzle.config.ts
    // Source: https://neon.com/docs/guides/drizzle
    import 'dotenv/config'
    import { defineConfig } from 'drizzle-kit'

    export default defineConfig({
      schema: './src/db/schema.ts',
      out: './drizzle',
      dialect: 'postgresql',
      dbCredentials: {
        url: process.env.DATABASE_URL!,
      },
    })
    ```

    If `.env.local` does not exist yet, create it from `.env.local.example`:

    Then pause and ask the user to fill in their actual `DATABASE_URL` from their Neon console.

    Once `DATABASE_URL` is confirmed present in `.env.local`, run:

    ```bash
    # Step 1: Generate SQL migration files from schema
    npx drizzle-kit generate
    # This creates ./drizzle/XXXX_init.sql (or similar name) — inspect it to verify correctness

    # Step 2: Apply migrations to Neon
    npx drizzle-kit migrate
    # This runs the SQL against your Neon database
    ```

    After migration, verify the tables exist by running Drizzle Studio briefly:
    ```bash
    npx drizzle-kit studio
    # Opens a browser UI at localhost:4983 showing your Neon tables
    # Confirm 'submissions' and 'evaluation_results' tables appear
    # Ctrl+C to stop studio after verification
    ```

    **If migration fails with "relation already exists":** The database already has these tables from a previous attempt. Run `npx drizzle-kit migrate` again — it will skip already-applied migrations. If truly stuck, inspect `drizzle/__drizzle_migrations` table in Neon console.

    **If migration fails with "DATABASE_URL is not set":** Verify `.env.local` exists, contains the correct Neon connection string, and `drizzle.config.ts` has `import 'dotenv/config'` at the top.

    **Critical: Do NOT use `npx drizzle-kit push` in production.** Use `generate` + `migrate` to maintain a migration history. `push` is for rapid prototyping only and skips the migration audit trail.

    Add a `db:migrate` script to `package.json` for future use:
    ```json
    {
      "scripts": {
        "db:generate": "drizzle-kit generate",
        "db:migrate": "drizzle-kit migrate",
        "db:studio": "drizzle-kit studio"
      }
    }
    ```
